# CIRO Emergency Rescue System — Multi-Agent Simulation Report

This report presents the shared state schema, environment tools, and automated trace execution logs generated by the **Google Antigravity** multi-agent orchestration workflow.

---

## 1. Shared State Schema
The system maintains a unified dataclass-driven `SystemState` shared between all agents:

- **PatientRequest**: Ingests emergency location, triage severity, status (`pending`, `dispatched`, `en_route`, `admitted`), and ICU/ventilator requirements.
- **AmbulanceState**: Tracks dynamic coordinates, active patient ID, selected hospital destination, and route polyline waypoints.
- **Hospital**: Registers capacity for general beds, ICU beds, mechanical ventilators, and trauma support status.

---

## 2. Environment Tools
The pipeline utilizes three native tools to interface with the state and execute actions:
1. `get_distance(coord1, coord2)`: Calculates Haversine distance and route ETAs.
2. `check_hospital_resources(hospital_id)`: Queries active bed, ICU, and ventilator capacity.
3. `reserve_bed(hospital_id, resource_type)`: Dynamically reserves a unit and decrements active registry counts.
